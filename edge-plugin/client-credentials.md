# Client Credentials

## Prerequisites

Before working with client credentials you should be familiar with the general plugin concepts described in the [Edge Plugin Overview](overview.md).

## Overview

The `ClientCredentials` node type generates OAuth-style client credentials (a client name and client secret) from the NativeEdge orchestrator, and retrieves the orchestrator's TLS certificate. These credentials and certificate are used by other components (applications, services, or scripts) that need to authenticate against the orchestrator API programmatically.

A typical use case is provisioning an edge application that must call back to the orchestrator -- the application needs a client identity and the orchestrator's TLS certificate to establish a trusted connection.

---

## Node Type

**Full type name:** `dell.nodes.nativeedge.template.ClientCredentials`
**Derived from:** `dell.nodes.Root`

### Properties

The properties for this node type are defined by the `client_credentials` DSL definition.

| Property | Type | Required | Default | Description |
|---|---|---|---|---|
| `client_name` | string | No | -- | The client name generated by the orchestrator. Typically left empty at deployment time and populated by the `precreate` operation. Can be pre-set if you want to reuse an existing client. |
| `client_secret` | string | No | -- | The client secret generated by the orchestrator. Typically left empty at deployment time and populated by the `precreate` operation. |
| `certificate` | `dell.datatypes.OrchestratorCertificate` | No | -- | The TLS certificate of the orchestrator. Populated by the `create` operation. |

### Lifecycle Operations

All operations live under the `dell.interfaces.lifecycle` interface.

| Operation | Implementation | Description |
|---|---|---|
| `precreate` | `edge.nativeedge_plugin.tasks.generate_client_secret` | Generates a new client name and client secret pair from the orchestrator's identity provider. The generated values are stored as runtime attributes on the node instance. |
| `create` | `edge.nativeedge_plugin.tasks.get_certificate` | Retrieves the orchestrator's TLS certificate and stores it as a runtime attribute. |

### Execution Flow

1. **precreate** runs first and generates the client credentials. After this step, the node's runtime properties will contain the `client_name` and `client_secret`.
2. **create** runs next and fetches the orchestrator's TLS certificate. After this step, the node's runtime properties will also contain the `certificate` data.

Other nodes in the blueprint can then access these values using `get_attribute`:

```yaml
{ get_attribute: [client_creds, client_name] }
{ get_attribute: [client_creds, client_secret] }
{ get_attribute: [client_creds, certificate] }
```

## Data Types

### OrchestratorCertificate

**Full type name:** `dell.datatypes.OrchestratorCertificate`

Represents a TLS certificate issued by or associated with the NativeEdge orchestrator.

| Property | Type | Required | Default | Description |
|---|---|---|---|---|
| `id` | string | No | -- | Unique identifier of the certificate. |
| `certText` | string | No | -- | The full certificate text (PEM-encoded content). |
| `expiration` | string | No | -- | Expiration date and time in ISO 8601 format (e.g., `2025-12-18T08:30:02Z`). |
| `format` | string | No | -- | Certificate format (e.g., `.PEM`). |
| `issuedBy` | string | No | -- | The certificate authority or entity that issued the certificate. |
| `issuedOn` | string | No | -- | Issue date and time in ISO 8601 format (e.g., `2024-12-18T08:30:02Z`). |
| `name` | string | No | -- | Human-readable name of the certificate. |
| `purpose` | string | No | -- | The intended purpose of the certificate. |
| `type` | string | No | -- | Certificate type identifier (e.g., `EO-UI`). |
| `valid` | string | No | -- | The date from which the certificate is valid, in ISO 8601 format. |

## Blueprint Example

```yaml
tosca_definitions_version: dell_1_0

imports:
  - dell/types/types.yaml
  - plugin:edge-plugin

node_templates:

  client_creds:
    type: dell.nodes.nativeedge.template.ClientCredentials

  my_application:
    type: dell.nodes.Root
    interfaces:
      dell.interfaces.lifecycle:
        create:
          implementation: scripts/configure_app.py
          executor: central_deployment_agent
          inputs:
            orchestrator_client_name: { get_attribute: [client_creds, client_name] }
            orchestrator_client_secret: { get_attribute: [client_creds, client_secret] }
            orchestrator_cert: { get_attribute: [client_creds, certificate, certText] }
    relationships:
      - type: dell.relationships.depends_on
        target: client_creds
```

### Explanation

1. **client_creds** -- Declares a `ClientCredentials` node with no explicit properties. The `precreate` operation generates a client name and secret, and the `create` operation retrieves the TLS certificate.
2. **my_application** -- A downstream node that needs to authenticate against the orchestrator:
   - It reads the generated `client_name`, `client_secret`, and the certificate's `certText` using `get_attribute`.
   - These values are passed as inputs to a configuration script.
3. The **relationship** ensures credentials are generated before the application node is created.
